## 概述

多角色权限管理模块为系统提供完整的用户身份与权限体系，支持四种角色（管理员、棚主、员工、专家）的注册登录和权限控制。棚主可自主管理名下员工，按大棚和功能两个维度分配权限。系统支持五级地区地址（省-市-区/县-乡镇-村），管理员可在Web端进行用户管理、角色管理和地区统计。

**覆盖端**：APP端（Android）+ Web管理端（Vue 3）

**使用角色**：
- 管理员(ADMIN)：仅Web端，管理用户/账号/角色/地区统计
- 棚主(OWNER)：APP端+Web端，管理自己员工和权限
- 员工(WORKER)：仅APP端，查看自己权限
- 专家(EXPERT)：仅Web端，专家工作台

## 用户故事

### US-001 用户注册与登录

> 作为大棚种植户，我希望注册一个棚主账号并登录系统，以便使用智慧大棚的各项功能。

**验收条件**：
- 注册时选择角色（棚主/员工，APP端无管理员选项）
- 注册时填写基本信息（手机号、密码、地区信息）
- 地区信息支持五级地址选择（省-市-区/县-乡镇-村）
- 登录支持手机号+密码方式
- 登录后根据角色显示对应的功能模块

### US-002 棚主 - 员工管理

> 作为大棚棚主，我手下有帮忙干活的员工，我希望在APP中创建员工账号并管理他们。

**验收条件**：
- 支持创建员工账号（手机号+姓名）
- 员工列表展示所有名下员工
- 支持删除员工（员工数据保留，解除归属关系）
- 员工只能归属一个棚主，不可切换
- 支持在Web端进行更方便的员工管理

### US-003 棚主 - 员工权限分配

> 作为大棚棚主，我希望控制每个员工能访问哪些大棚、使用哪些功能。

**验收条件**：
- 权限配置包含两个维度：大棚授权 + 功能授权
- 大棚授权：勾选员工可访问的大棚
- 功能授权：勾选员工可使用的功能（查看数据/控制设备/拍照诊断/求助专家/查看预警/查看历史）
- Web端提供权限矩阵可视化配置界面（大棚×功能交叉授权）
- 权限修改后立即生效

### US-004 棚主 - 员工权限查看与修改

> 作为大棚棚主，我希望随时查看和调整员工的权限。

**验收条件**：
- 员工列表中可查看每个员工的当前权限
- 点击员工进入权限详情页
- 可随时修改权限并保存
- 权限变更记录保存到操作日志

### US-005 员工 - 查看自己权限

> 作为员工，我希望了解自己能访问哪些大棚和使用哪些功能，避免越权操作。

**验收条件**：
- 个人中心显示当前权限范围
- 列出可访问的大棚列表
- 列出可使用的功能列表
- 权限变更后有通知提醒

### US-006 地区信息填写

> 作为大棚棚主，注册时需要填写大棚所在地区，以便平台按地区统计和就近分配专家。

**验收条件**：
- 支持五级地址选择：省 → 市 → 区/县 → 乡镇 → 村
- 地址选择器为级联选择，逐级展开
- 注册后可在个人中心修改地区信息
- 地区数据用于Web端地区分布统计

### US-007 管理员 - 用户管理

> 作为平台管理员（Web端），我希望管理所有用户账号，包括创建、查看、编辑和状态管理。

**验收条件**：
- 用户列表展示所有用户（棚主/员工/专家）
- 支持创建各类角色账号
- 支持编辑用户基本信息
- 支持启用/禁用用户账号
- 支持按角色筛选用户

### US-008 管理员 - 角色与关系管理

> 作为平台管理员，我希望查看平台的角色分布和棚主-员工的从属关系。

**验收条件**：
- 角色管理页面展示四种角色及其说明
- 支持创建/编辑专家账号（专业领域设置）
- 查看棚主-员工关系（哪个员工归属哪个棚主）
- 用户地区分布统计（按省/市/区县）

### US-009 管理员 - 平台数据概览

> 作为平台管理员，我希望查看平台的整体运营数据，但不干预用户的决策和操作。

**验收条件**：
- 展示平台用户总数、大棚总数、设备总数
- 展示各地区用户分布图
- 展示平台活跃度统计
- 管理员无权限操作用户的设备、查看非公开数据
- 平台定位：提供平台，不干预用户决策

## 验收标准

### 功能验收

- [ ] 注册流程支持角色选择（棚主/员工）
- [ ] 登录后根据角色展示对应功能模块
- [ ] 棚主可创建/删除员工账号
- [ ] 员工权限分配支持大棚维度和功能维度
- [ ] 权限修改后实时生效
- [ ] 员工只能看到被授权的大棚和功能
- [ ] 五级地址选择器数据完整且级联正确
- [ ] 管理员可启用/禁用用户
- [ ] 棚主-员工关系查看正确
- [ ] 地区分布统计数据准确

### 安全验收

- [ ] JWT token过期后自动登出
- [ ] 员工无法访问未授权的大棚数据
- [ ] 员工无法使用未授权的功能
- [ ] API接口层有权限校验（不可前端绕过）
- [ ] 密码加密存储（BCrypt）
- [ ] 管理员无法操作用户设备（平台无管理权）

### 性能验收

- [ ] 登录响应时间 < 1秒
- [ ] 权限校验不影响API响应时间（< 50ms开销）

## 备注

1. **角色定义**（v3.1修正后）：
   | 角色 | 端 | 说明 |
   |------|-----|------|
   | 管理员(ADMIN) | 仅Web端 | 管理用户/账号/平台总览，不干预用户决策 |
   | 棚主(OWNER) | APP + Web | 大棚所有者，管理自己大棚和员工 |
   | 员工(WORKER) | 仅APP | 棚主名下员工，权限由棚主分配，单归属 |
   | 专家(EXPERT) | 仅Web | 平台入驻专家，通过专家工作台处理求助 |

2. **权限模型**：
   ```
   用户权限 = 角色基础权限 ∩ 大棚授权 ∩ 功能授权
   ```
   - 角色基础权限：定义每种角色能做什么的默认权限
   - 大棚授权：指定员工可访问哪些大棚（数据隔离）
   - 功能授权：指定员工可使用哪些功能（操作粒度）

3. **功能权限项**（可分配）：
   - `VIEW_DATA`：查看数据
   - `CONTROL_DEVICE`：控制设备
   - `PHOTO_DIAGNOSIS`：拍照诊断
   - `ASK_EXPERT`：求助专家
   - `VIEW_ALERT`：查看预警
   - `VIEW_HISTORY`：查看历史

4. **员工单归属规则**（v3.1修正）：
   - 员工只能归属一个棚主
   - 不存在切换棚主的情况
   - 棚主删除员工后，员工账号保留但失去归属关系
   - 被删除的员工需重新被邀请才能归属新棚主

5. **五级地址数据**：地区数据为静态数据，以JSON文件形式存储在后端，支持级联查询。数据来源可使用国家统计局行政区划数据。

6. **JWT Token设计**：
   ```
   {
     user_id, role, greenhouse_ids[], permissions[],
     exp, iat
   }
   ```
