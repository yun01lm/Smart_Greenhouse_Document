## 概述

AI智能问答模块基于检索增强生成（RAG）架构，结合大语言模型和专业知识库，为用户提供农业种植相关的智能问答服务。用户可通过文字或语音（支持方言识别）输入问题，系统从知识库中检索相关内容，由DeepSeek大模型生成专业回答，并标注引用来源，确保答案的可溯源性。回答结果支持语音播报（TTS）。

**覆盖端**：APP端（Android）+ Web管理端（知识库测试）

**使用角色**：
- 棚主(OWNER)：使用AI问答功能
- 员工(WORKER)：使用AI问答功能
- 管理员(ADMIN)：在Web端测试RAG问答效果

## 用户故事

### US-001 文字输入问答

> 作为大棚棚主，我遇到种植问题（如"番茄叶片发黄是什么原因"），希望通过文字输入问题并获得专业的回答。

**验收条件**：
- 提供文字输入框，支持多行输入
- 发送问题后显示加载状态
- 回答以对话气泡形式展示
- 支持追问（上下文连续对话）

### US-002 语音输入问答（方言识别）

> 作为大棚棚主（河北地区用户），我希望用河北话语音提问，系统能识别并给出回答。

**验收条件**：
- 支持点击语音按钮开始录音
- 录音过程中显示波形动画
- 录音结束后自动识别转文字（支持河北方言）
- 转文字结果可编辑修正后发送
- 初期使用讯飞ASR API，后期预留Whisper微调接口

### US-003 知识库检索增强生成

> 作为大棚棚主，我希望AI的回答基于专业的农业知识，而不是通用的网络信息。

**验收条件**：
- 回答内容基于知识库中的专业文档生成
- 当知识库中有相关内容时优先使用知识库信息
- 当知识库无相关内容时，基于DeepSeek通用知识回答并标注"基于通用知识"
- 回答专业、准确、实用

### US-004 答案引用来源展示

> 作为大棚棚主，我希望看到AI回答的依据来源，以便验证信息的可靠性。

**验收条件**：
- 每条回答下方展示引用来源列表
- 引用来源包含：文档名称、相关段落摘要
- 点击来源可查看原文内容
- 无知识库来源时标注"基于通用知识"

### US-005 问答历史记录

> 作为大棚棚主，我希望查看之前的问答记录，方便回顾和查阅。

**验收条件**：
- 问答记录以对话列表形式展示
- 每条对话显示问题摘要和时间
- 点击可查看完整对话内容
- 支持删除不需要的对话记录

### US-006 语音播报回答

> 作为大棚棚主，我在忙农活时不便看手机，希望系统能语音播报AI的回答。

**验收条件**：
- 回答消息旁提供"播放"按钮
- 点击播放按钮，使用TTS语音播报回答内容
- 支持暂停/继续播放
- 播放完成后按钮恢复默认状态

## 验收标准

### 功能验收

- [ ] 文字输入问答功能正常，支持多轮对话（上下文保持至少5轮）
- [ ] 语音输入可正常录音并转文字
- [ ] 河北方言基础词汇识别准确率 > 85%
- [ ] RAG检索结果与问题相关性高（前3条结果包含相关信息）
- [ ] 回答引用来源正确且可点击查看原文
- [ ] 问答历史记录完整保存，支持回溯
- [ ] 语音播报功能正常，播报内容与文字回答一致

### 性能验收

- [ ] 文字问答响应时间 < 10秒（含RAG检索+LLM生成）
- [ ] 语音识别响应时间 < 5秒
- [ ] 语音播报开始播放延迟 < 2秒

### AI质量验收

- [ ] 回答准确率 > 80%（基于知识库内容的问答）
- [ ] 回答与问题相关性 > 85%
- [ ] 无幻觉内容（虚构不存在的信息）

## 备注

1. **RAG架构流程**：
   ```
   用户提问 → 向量检索(Chroma) → 获取Top-K相关文档片段
   → 构建Prompt(系统提示+上下文+问题) → DeepSeek生成回答
   → 标注引用来源 → 返回结果
   ```

2. **语音识别方案切换**：通过策略模式实现 `SpeechRecognitionProvider` 接口，初期实现 `XunfeiASRProvider`，后期实现 `WhisperProvider`。方言语料收集阶段同时作为Whisper微调的训练数据。

3. **支持的方言**：初期聚焦河北方言（申报书团队所在地），后期可扩展其他方言。语音识别API选择需确认对河北方言的支持程度。

4. **问答上下文管理**：每轮对话保留最近5轮问答作为上下文。对话以会话(Session)为单位管理，用户可开启新会话重置上下文。

5. **知识库覆盖范围**（初期）：
   - 常见大棚蔬菜病虫害防治（番茄、黄瓜、辣椒、茄子等）
   - 大棚环境调控技术（温湿度、光照、CO₂管理）
   - 水肥管理技术
   - 栽培技术要点

6. **成本控制**：DeepSeek API价格极低（约0.001元/1K tokens），但仍需在Web端系统监控模块统计API调用量和费用。

7. **申报书对应**：本模块对应申报书五大研究内容之"知识库+方言AI问答"。
