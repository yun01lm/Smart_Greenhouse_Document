## 概述

作物生长周期管理模块为每个大棚的每轮种植建立完整的生长周期记录，从种植日期到收获日期的全过程管理。该模块支持种植记录的创建、阶段推进、完成收获，以及生长时间线的完整追溯。通过关联长势评估记录和环境预警记录，为用户提供作物生长全过程的可视化追溯。

**覆盖端**：APP端（Android）

**使用角色**：
- 棚主(OWNER)：管理自己大棚的种植周期
- 员工(WORKER)：查看被授权大棚的种植周期（可能限制编辑权限）

## 用户故事

### US-001 创建种植记录

> 作为大棚棚主，当我开始一轮新种植时，希望在APP中创建一个种植记录，记录作物名称、品种和种植日期。

**验收条件**：
- 支持选择大棚后创建新的种植记录
- 输入/选择作物名称（如番茄、黄瓜等）
- 输入作物品种名称
- 选择种植日期（默认当天）
- 可选择作物类型对应的默认生长阶段模板
- 创建成功后显示在生长周期列表中

### US-002 生长周期列表查看

> 作为大棚棚主，我希望查看每个大棚的种植历史，包括当前进行中和已完成的种植周期。

**验收条件**：
- 按大棚筛选显示种植周期列表
- 每个周期显示：作物名称、品种、种植日期、当前阶段、已生长天数
- 区分进行中和已完成的周期（不同颜色/标签）
- 当前进行中的周期置顶显示

### US-003 生长阶段管理

> 作为大棚棚主，我希望查看和管理作物的生长阶段，系统可根据日期自动估算，也支持手动修正。

**验收条件**：
- 展示当前生长阶段和进入该阶段的日期
- 系统根据种植日期和预设阶段时长自动估算当前阶段
- 支持手动推进到下一阶段
- 支持手动修正阶段（回退或跳过）
- 阶段变更记录保存到历史

### US-004 生长时间线视图

> 作为大棚棚主，我希望按时间轴查看作物从种植到现在的完整生长历程。

**验收条件**：
- 时间轴从种植日期开始到当前日期（或收获日期）
- 时间轴上标注各生长阶段的分界点
- 每个阶段节点展示该阶段的长势评估摘要
- 时间轴上同时标注预警事件（颜色区分）
- 点击节点可查看详细信息
- 支持缩放和滚动浏览

### US-005 生长周期完成

> 作为大棚棚主，当作物成熟收获时，希望标记该种植周期为已完成。

**验收条件**：
- 提供"完成种植（收获）"操作
- 完成前确认提示
- 完成后周期状态变为"已完成"
- 记录收获日期
- 已完成周期保留查看但不可再修改阶段

### US-006 生长数据关联查看

> 作为大棚棚主，查看种植周期时，希望能关联看到该周期的长势评估和病虫害诊断记录。

**验收条件**：
- 周期详情页展示关联的长势评估记录列表
- 展示关联的病虫害诊断记录列表
- 展示关联的环境预警记录列表
- 支持点击跳转到对应详情

## 验收标准

### 功能验收

- [ ] 可成功创建种植记录，所有必填字段验证通过
- [ ] 生长周期列表正确展示并按大棚筛选
- [ ] 进行中和已完成周期区分明确
- [ ] 已生长天数计算准确
- [ ] 生长阶段自动估算合理（基于预设阶段时长）
- [ ] 手动修正阶段功能正常，记录变更历史
- [ ] 生长时间线正确展示阶段分界和关联事件
- [ ] 完成收获后周期状态变更正确
- [ ] 关联数据（长势评估/诊断/预警）正确展示

### 性能验收

- [ ] 生长时间线加载时间 < 2秒（含关联数据）
- [ ] 周期列表分页加载，每页20条

### 数据完整性验收

- [ ] 阶段变更历史完整记录（何时、从何阶段变为何阶段、操作方式）
- [ ] 收获后周期不可再修改阶段
- [ ] 删除周期需确认，关联数据不级联删除

## 备注

1. **API端点设计**（对应功能点规划清单第四章）：
   | 方法 | 端点 | 说明 |
   |------|------|------|
   | GET | /api/crop-cycles | 生长周期列表（按大棚过滤） |
   | POST | /api/crop-cycles | 创建种植记录 |
   | GET | /api/crop-cycles/{id} | 周期详情 |
   | PUT | /api/crop-cycles/{id} | 更新（推进阶段等） |
   | PUT | /api/crop-cycles/{id}/complete | 标记完成（收获） |
   | GET | /api/crop-cycles/{id}/timeline | 生长时间线（关联长势评估+预警记录） |

2. **crop_cycles表关键字段**：
   - id, greenhouse_id, crop_name, variety
   - plant_date（种植日期）
   - current_stage（当前阶段：苗期/生长期/开花期/结果期）
   - stage_start_date（当前阶段开始日期）
   - status（ACTIVE/COMPLETED）
   - harvest_date（收获日期，完成时填写）
   - created_at, updated_at

3. **阶段自动估算逻辑**：
   - 系统预设各作物类型的默认阶段时长（可配置）
   - 根据种植日期和当前日期计算预计所处阶段
   - 自动更新current_stage和stage_start_date
   - 用户可手动修正（实际生长可能快于或慢于预设）

4. **生长时间线数据结构**：
   ```
   timeline: [
     {type: "plant", date: "2026-03-01", description: "种植番茄"},
     {type: "stage_change", date: "2026-03-31", description: "进入生长期"},
     {type: "growth_assessment", date: "2026-04-15", summary: "株高35cm, 叶色健康"},
     {type: "alert", date: "2026-04-20", level: "warning", description: "温度过高预警"},
     {type: "diagnosis", date: "2026-05-01", description: "蚜虫-已处理"},
     {type: "stage_change", date: "2026-05-15", description: "进入开花期"},
     ...
   ]
   ```

5. **支持作物类型**（初期）：
   - 番茄、黄瓜、辣椒、茄子、草莓、西瓜、甜瓜
   - 每种作物有独立的默认阶段时长配置

6. **数据关联**：长势评估记录（growth_assessments表）和诊断记录（diagnosis_records表）增加crop_cycle_id外键，与生长周期关联。
