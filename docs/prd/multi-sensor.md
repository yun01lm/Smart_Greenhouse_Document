## 概述

多组传感器管理模块支持大棚内3-5组传感器设备的部署和管理。每组传感器由独立ESP32主控，通过WiFi+MQTT上报数据（消息含group_id标识）。该模块提供传感器组的CRUD管理、区域标注（东侧/西侧/入口等）、设备分组配置、组状态监控等功能，支撑数据看板中的多组切换查看、多组对比、大棚聚合视图和预警传感器组定位。

**覆盖端**：Web管理端（Vue 3）+ APP端（数据查看）

**使用角色**：
- 棚主(OWNER)：在Web端管理自己大棚的传感器组
- 管理员(ADMIN)：在Web端管理全部设备分组

## 用户故事

### US-001 传感器组创建与配置

> 作为大棚棚主，我希望在Web端创建传感器组，配置组名、区域标注和组内设备。

**验收条件**：
- 支持为大棚创建新的传感器组
- 配置组名（如"东侧-组1"、"育苗区"等）
- 配置区域标注（选择预定义区域：东侧/西侧/南侧/北侧/入口/中央/自定义）
- 将已注册的ESP32设备分配到传感器组

### US-002 设备分组管理

> 作为大棚棚主，我希望将设备编入不同的传感器组，并查看各组的设备组成。

**验收条件**：
- 展示已注册但未分组的设备列表
- 支持将设备分配到指定传感器组
- 支持从传感器组中移除设备
- 每组可包含1个ESP32主控 + N个传感器模块

### US-003 传感器组区域标注

> 作为大棚棚主，我希望为每组传感器设置区域标注名称，以便在数据展示时能知道数据来源。

**验收条件**：
- 支持选择预定义区域名称
- 支持自定义区域名称
- 区域名称在数据看板中跟随数据展示
- 区域名称在预警通知中显示

### US-004 传感器组状态监控

> 作为大棚棚主，我希望查看各传感器组的在线/离线状态。

**验收条件**：
- 传感器组列表展示各组在线状态
- 显示最后数据上报时间（心跳）
- 离线组红色标识并显示离线时长
- 支持按状态筛选

### US-005 传感器配置

> 作为管理员/棚主，我希望配置各传感器组的传感器参数。

**验收条件**：
- 配置传感器类型（温度/湿度/光照/CO₂/O₂/土壤温度/土壤湿度/EC/N/P/K）
- 配置I2C地址（多传感器共享总线时区分）
- 配置采样频率（默认30秒一次）
- 配置传感器校准参数

### US-006 设备固件管理

> 作为管理员，我希望查看和管理ESP32设备的固件版本。

**验收条件**：
- 展示各ESP32设备的固件版本号
- 支持记录固件更新历史
- 此功能为可选功能，初期可简化

## 验收标准

### 功能验收

- [ ] 支持为大棚创建3-5组传感器组
- [ ] 传感器组支持自定义命名和区域标注
- [ ] 设备可正确分配到指定传感器组
- [ ] 传感器组状态监控准确（在线/离线/心跳时间）
- [ ] 传感器配置修改后1分钟内生效
- [ ] 数据看板中可正确切换不同传感器组查看数据
- [ ] 多组对比视图可同时展示最多5组数据
- [ ] 聚合视图的平均值/最高最低值计算正确

### 性能验收

- [ ] MQTT消息含group_id标识，不影响消息处理性能
- [ ] 多组数据对比查询响应时间 < 2秒
- [ ] 传感器组状态刷新间隔 < 10秒

### 硬件验收

- [ ] 每组ESP32独立WiFi连接稳定
- [ ] 每组传感器数据上报成功率 > 99%
- [ ] 传感器组之间无相互干扰

## 备注

1. **传感器组硬件架构**：
   ```
   大棚A
   ├── 传感器组1 (东侧) — ESP32 #1 + 传感器模块
   ├── 传感器组2 (西侧) — ESP32 #2 + 传感器模块
   ├── 传感器组3 (入口) — ESP32 #3 + 传感器模块
   ├── 传感器组4 (中央) — ESP32 #4 + 传感器模块
   └── 传感器组5 (北侧) — ESP32 #5 + 传感器模块
   ```

2. **MQTT Topic设计（含分组）**：
   - 数据上报：`greenhouse/{greenhouse_id}/group/{group_id}/sensor`
   - 心跳：`greenhouse/{greenhouse_id}/group/{group_id}/heartbeat`
   - 控制指令：`greenhouse/{greenhouse_id}/group/{group_id}/device/{device_id}/control`

3. **InfluxDB数据模型**：
   ```
   measurement: sensor_data
   tags: greenhouse_id, group_id, group_name, region
   fields: temperature, humidity, light, co2, o2, soil_temp, soil_humidity, soil_ec, soil_n, soil_p, soil_k
   timestamp:上报时间
   ```

4. **推荐组数**：建议3组起步（东/西/入口），最大5组。组数越多数据越精确，但硬件成本和维护复杂度也增加。

5. **区域标注预设**：
   - 东侧、西侧、南侧、北侧
   - 入口处、中央区
   - 育苗区、生长区、结果区
   - 支持自定义

6. **心跳机制**：每组ESP32每30秒发送一次心跳，后端超过60秒未收到心跳标记为离线。
