## 概述

环境预警中心负责监控大棚环境参数，当数据超出正常范围或呈现异常模式时，自动生成分级预警报告并向用户推送通知。该模块结合天气预报进行环境趋势预判，利用LSTM时序预测模型预测环境因子短期变化，并支持用户自定义预警阈值，以适应不同作物品种和生长阶段的需求。

**覆盖端**：APP端（Android）+ Web管理端（Vue 3）

**使用角色**：
- 棚主(OWNER)：查看自己大棚预警、自定义预警阈值
- 员工(WORKER)：查看被��权大棚的预警
- 管理员(ADMIN)：查看全部大棚预警
- 专家(EXPERT)：查看被授权大棚预警（7天有效期内）

## 用户故事

### US-001 分级预警展示

> 作为大棚棚主，我希望系统能根据环境参数异常程度自动分级预警（普通/警告/严重），并在APP中以醒目的方式展示，以便我及时采取应对措施。

**验收条件**：
- 预警分为三级：普通（蓝色）、警告（橙色）、严重（红色）
- 预警卡片显示：触发参数、当前值、阈值、触发时间、关联传感器组
- 预警列表按严重程度和时间排序
- 支持按级别筛选预警

### US-002 预警推送通知

> 作为大棚棚主，即使APP在后台或手机锁屏，我也希望能收到预警推送通知，不错过任何异常情况。

**验收条件**：
- APP后台时通过系统通知栏推送预警
- 点击通知可直接跳转到预警详情页
- 支持设置通知免打扰时段
- 通知内容包含预警级别、参数名称、当前值和阈值

### US-003 预警历史记录

> 作为大棚管理人员，我希望查看历史预警记录，了解大棚过去发生的异常情况。

**验收条件**：
- 支持按时间范围查询历史预警
- 支持按预警级别筛选
- 支持按参数类型筛选
- 展示每条预警的详细信息（参数、值、阈值、触发时间、恢复时间）

### US-004 天气预报结合趋势预判

> 作为大棚棚主，我希望看到结合天气预报的环境趋势预判，以便提前规划大棚管理措施。

**验收条件**：
- 展示未来24-72小时天气预报数据
- 结合天气预报和当前环境数据，展示环境趋势预判
- 预判可能触发预警的时间点，提前标注
- 天气数据来源：和风天气免费API

### US-005 环境参数短期预测

> 作为大棚管理人员，我希望看到基于LSTM模型的环境参数短期预测曲线，以便提前了解未来几小时的环境变化。

**验收条件**：
- 展示未来30分钟至2小时的温度/湿度预测曲线
- 预测值与实际值对比展示
- 标注预测置信区间
- 初期使用统计方法，后期替换为LSTM模型

### US-006 异常模式识别

> 作为大棚棚主，当传感器数据呈现病害高发特征模式时，我希望系统能自动识别并预警。

**验收条件**：
- 系统内置常见病害高发环境特征模式库
- 当环境数据匹配异常模式时自动触发预警
- 预警附带可能的病害类型说明
- 支持管理员在Web端管理异常模式库

### US-007 预警关联传感器组定位

> 作为大棚棚主，大棚内有多组传感器，当预警触发时，我希望立即知道是哪组传感器（哪个区域）触发了预警。

**验收条件**：
- 预警详情明确显示触发传感器组名称和区域标注
- 支持按传感器组筛选预警记录
- 预警通知中直接显示传感器组区域名称

### US-008 自定义预警阈值

> 作为大棚棚主，不同作物不同生长阶段对环境的耐受范围不同，我希望能自己设置各参数的最高最低预警值，而不是使用系统默认值。

**验收条件**：
- 支持为每个大棚设置各参数的最高预警值和最低预警值
- 可设置参数：温度、湿度、光照、CO₂、O₂、土壤温度、土壤湿度、土壤EC
- 自定义阈值优先级高于系统默认阈值
- 修改阈值后实时生效，无需重启
- 支持一键恢复系统默认阈值

## 验收标准

### 功能验收

- [ ] 三级预警（普通/警告/严重）分类正确，颜色标识清晰
- [ ] 预警通知在APP后台和锁屏状态下可正常推送
- [ ] 点击通知可跳转到对应预警详情
- [ ] 预警历史记录查询支持按时间/级别/参数/传感器组筛选
- [ ] 天气预报数据每日至少更新4次
- [ ] 环境趋势预判准确率（预警提前量 > 15分钟）
- [ ] 异常模式库至少包含3种常见病害高发特征模式

- [ ] 自定义阈值优先级验证：用户设置值生效，系统默认值被覆盖

- [ ] 预警从数据到达→判断→通知推送延迟 < 5秒
- [ ] Web端预警规则配置界面操作响应 < 1秒
- [ ] 异常模式匹配不阻塞正常数据流

### 兼容性验收

- [ ] 支持Android 8.0及以上版本的通知渠道
- [ ] 通知权限申请流程符合Android规范
- [ ] Web端支持Chrome/Firefox/Edge主流浏览器

## 备注

1. **预警等级定义**：
   - **普通**：参数超出设定阈值但 < 10%偏差
   - **警告**：参数超出阈值10%-20%偏差
   - **严重**：参数超出阈值 > 20%偏差，或匹配异常模式

2. **LSTM预测模型**：初期使用统计方法（移动平均+趋势外推）实现预测，Phase 4阶段在Colab上训练LSTM模型替换。需在代码中预留Provider接口。

3. **自定义阈值表结构**（user_alert_thresholds）：
   - 关联字段：user_id、greenhouse_id、parameter_type
   - 阈值字段：min_value、max_value
   - 优先级：用户自定义 > 系统默认

4. **通知免打扰**：支持按时间段设置（如22:00-06:00），该时段严重级别预警仍推送，普通和警告延迟到免打扰结束后推送。

5. **异常模式库**：以JSON格式存储特征模式，包含参数组合条件（如"温度>30°C AND 湿度>85%持续2小时"），匹配逻辑在后端预警引擎中实现。

6. **申报书对应**：本模块对应申报书五大研究内容之"时序预测环境预警"。
