## 概述

实时数据看板是智慧大棚AIoT系统的核心展示模块，为用户提供大棚环境数据的直观可视化界面。该模块以仪表盘、趋势曲线等形式呈现空气温湿度、光照强度、CO₂/O₂浓度、土壤温湿度等关键环境参数的实时数值与历史变化趋势，支持多棚切换查看和多组传感器数据对比。

**覆盖端**：APP端（Android）

**使用角色**：
- 棚主(OWNER)：查看自己名下全部大棚数据
- 员工(WORKER)：查看被授权大棚的数据
- 专家(EXPERT)：查看被授权大棚的7天有效期内数据

## 用户故事

### US-001 仪表盘实时查看

> 作为大棚棚主，我希望打开APP后能在一个仪表盘页面上看到当前大棚的关键环境数值（温度、湿度、光照、CO₂、O₂、土壤数据），以便快速了解大棚环境状况。

**验收条件**：
- 页面展示当前选中大棚的空气温度、空气湿度、光照强度、CO₂浓度、O₂浓度
- 页面展示土壤温度、土壤湿度、土壤EC值、土壤N/P/K
- 各数值附带正常/异常状态标识（颜色区分）
- 数据通过WebSocket实时推送，自动刷新无需手动操作

### US-002 趋势曲线图查看

> 作为大棚棚主，我希望查看不同环境参数的趋势曲线图，并可切换时间范围（1小时/24小时/7天/30天），以便分析环境变化规律。

**验收条件**：
- 支持切换不同环境参数查看（温度/湿度/光照/CO₂/O₂/土壤各参数）
- 支持切换时间范围：1小时、24小时、7天、30天
- 曲线图支持缩放、平移交互
- 数据点超过阈值时曲线变色标记

### US-003 多棚切换

> 作为拥有多个大棚的棚主，我希望能在APP中快速切换查看不同大棚的数据，无需退出当前页面。

**验收条件**：
- 顶部提供大棚选择器，展示棚主名下所有大棚
- 切换大棚后仪表盘和曲线图同步更新
- 记录上次选择的大棚，下次打开默认显示

### US-004 历史数据回溯

> 作为大棚管理人员，我希望能够回溯查询任意时间段的历史传感器数据，以便分析历史环境状况。

**验收条件**：
- 支持选择起止日期查询历史数据
- 支持选择特定参数查询
- 查询结果以数据列表和曲线图两种形式展示
- 支持按小时/天粒度聚合

### US-005 多组传感器查看

> 作为大棚棚主，大棚内安装了3-5组传感器分布在不同区域，我希望能够选择特定传感器组查看其详细数据。

**验收条件**：
- 提供传感器组选择器（如"东侧-组1"、"西侧-组2"、"入口-组3"）
- 选择某组后，仪表盘和曲线图切换为该组数据
- 显示传感器组区域标注名称

### US-006 多组数据对比

> 作为大棚棚主，我希望能够同时查看多组传感器同一参数的对比曲线，以便直观了解不同区域的环境差异。

**验收条件**：
- 支持选择同一参数（如温度），同时展示各组传感器曲线
- 不同组曲线用不同颜色区分
- 支持最多5组同时对比
- 图例显示各组名称和区域标注

### US-007 大棚聚合数据视图

> 作为大棚棚主，我希望查看整个大棚的聚合环境数据（全棚平均值、最高最低值、区域热力图），以便快速把握大棚整体状况。

**验收条件**：
- 展示全棚各参数的平均值、最高值、最低值
- 提供区域热力图，以颜色深浅表示不同区域参数差异
- 聚合数据随实时数据自动更新

## 验收标准

### 功能验收

- [ ] 仪表盘正确展示10+项环境参数（空气温度、空气湿度、光照、CO₂、O₂、土壤温度、土壤湿度、EC、N、P、K）
- [ ] 趋势曲线图支持4个时间范围切换且数据准确
- [ ] 多棚切换响应时间 < 500ms
- [ ] 历史数据查询支持至少30天范围
- [ ] WebSocket推送延迟 < 2秒（局域网环境）
- [ ] 数据异常高亮标记与预警阈值规则一致
- [ ] 多组传感器切换后数据正确对应所选组
- [ ] 多组对比曲线图例清晰，颜色区分明显
- [ ] 聚合视图的平均值/最高最低值计算正确
- [ ] 区域热力图基于实际传感器位置渲染

### 性能验收

- [ ] APP端首屏加载时间 < 3秒
- [ ] 曲线图渲染500个数据点无明显卡顿
- [ ] WebSocket重连机制正常（断网后自动重连）
- [ ] 支持同时展示5组对比曲线不卡顿

### 兼容性验收

- [ ] 支持Android 8.0及以上版本
- [ ] 适配不同屏幕尺寸（手机/平板）
- [ ] 横竖屏切换数据展示正常

## 备注

1. **数据刷新策略**：APP端采用WebSocket推送+本地缓存策略。前台时实时推送更新，后台时暂停推送以节省流量和电量。
2. **离线缓存**：网络断开时展示缓存的最新数据，并在数据上方显示"数据更新时间"标签，网络恢复后自动重连。
3. **传感器组命名规范**：区域标注名称由棚主在Web端配置，支持自定义名称（如"育苗区-组1"、"结果区-组2"）。
4. **数据精度**：温度保留1位小数，湿度保留1位小数，光照保留整数，CO₂保留整数，O₂保留1位小数，土壤EC保留2位小数。
5. **图表库选型**：Android端推荐使用MPAndroidChart库实现趋势曲线图和仪表盘。
6. **申报书对应**：本模块对应申报书五大研究内容之"多参数传感网络与实时看板"。
