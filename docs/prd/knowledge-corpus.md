## 概述

知识库与语料管理模块为系统的RAG（检索增强生成）问答能力和病虫害诊断防治方案提供知识支撑。管理员通过Web端上传农业专业文档（Markdown/TXT/PDF格式），系统自动进行文档切片、向量化处理并存入Chroma向量数据库。同时提供方言语料管理功能，为后期Whisper方言识别模型微调积累训练数据。

**覆盖端**：Web管理端（Vue 3）

**使用角色**：
- 管理员(ADMIN)：管理知识库文档和方言语料

## 用户故事

### US-001 知识文档上传

> 作为平台管理员，我希望上传农业专业知识文档到系统中，丰富知识库内容。

**验收条件**：
- 支持上传Markdown(.md)、TXT(.txt)、PDF(.pdf)格式
- 单文件大小限制 < 20MB
- 上传时选择文档分类（病虫害防治/栽培技术/水肥管理/品种选择）
- 上传后显示处理状态（待处理→处理中→已完成→处理失败）
- 支持批量上传

### US-002 文档分类管理

> 作为平台管理员，我希望对知识库文档进行分类管理，方便组织和检索。

**验收条件**：
- 预设分类：病虫害防治、栽培技术、水肥管理、品种选择
- 支持创建自定义分类
- 文档列表支持按分类筛选
- 每个分类显示文档数量

### US-003 自动向量化索引

> 作为平台管理员，上传文档后系统应自动进行文档切片和向量化处理，无需手动操作。

**验收条件**：
- 上传后自动触发文档切片（按段落/固定长度切片）
- 切片后自动调用Embedding模型生成向量
- 向量自动存入Chroma向量数据库
- 处理完成后文档状态更新为"已完成"
- 处理失败时显示错误原因

### US-004 问答测试

> 作为平台管理员，上传文档后希望在Web端直接测试RAG问答效果，验证知识库内容的有效性。

**验收条件**：
- Web端提供问答测试输入框
- 输入问题后返回RAG生成的回答
- 展示检索到的相关文档片段
- 展示引用来源（文档名称+段落位置）
- 支持多轮测试

### US-005 文档列表与删除

> 作为平台管理员，我希望查看已上传的文档列表，并可以删除不需要的文档。

**验收条件**：
- 文档列表展示：文件名、分类、大小、上传时间、处理状态
- 支持按分类、状态筛选
- 支持搜索文档名称
- 删除文档时同步删除Chroma中的向量数据
- 删除前需确认

### US-006 方言语料管理

> 作为平台管理员，我希望管理系统中的方言语料数据，为方言识别模型训练积累素材。

**验收条件**：
- 支持上传语料文件（音频+标注文本）
- 语料列表展示：文件名、方言类型、上传时间、标注状态
- 支持查看语料标注内容
- 支持删除不需要的语料
- 标注格式：音频文件+对应文本标注

## 验收标准

### 功能验收

- [ ] 支持上传MD/TXT/PDF三种格式文档
- [ ] 上传后自动完成切片和向量化处理
- [ ] 处理完成后文档状态正确更新

- [ ] 文档删除后Chroma中向量数据同步删除
- [ ] 文档分类筛选功能正常
- [ ] 方言语料上传和管理功能正常

### 性能验收

- [ ] 文档切片+向量化处理时间：1MB文档 < 30秒
- [ ] 向量检索响应时间 < 1秒
- [ ] 支持同时处理最多5个文档

### 数据完整性验收

- [ ] 文档删除后向量数据完全清除（无残留）
- [ ] 文档更新（替换上传）时旧向量数据正确清理
- [ ] 处理失败的文档可重新触发处理

## 备注

1. **知识库内容范围**（初期优先级）：
   | 优先级 | 内容类别 | 说明 |
   |--------|----------|------|
   | P0 | 常见病虫害防治 | 番茄、黄瓜、辣椒、茄子等主要大棚蔬菜的病虫害识别与防治 |
   | P1 | 大棚环境调控 | 温湿度/光照/CO₂管理技术 |
   | P2 | 水肥管理技术 | 滴灌方案、肥料配比 |
   | P3 | 栽培技术要点 | 育苗/定植/整枝/授粉技术 |
   | P4 | 品种选择指南 | 不同地区/季节的品种推荐 |

2. **文档切片策略**：
   - 按段落边界切片（以空行/标题为分割点）
   - 每片大小控制在500-1000字符
   - 相邻切片保留200字符重叠（防止信息断裂）
   - 切片元数据记录：文档ID、分类、段落位置

3. **Chroma Collection设计**：
   - Collection名称：`greenhouse_knowledge`
   - 元数据字段：`doc_id`, `category`, `chunk_index`, `source_file`
   - Embedding维度：根据所选Embedding模型决定

4. **方言语料管理**：
   - 支持方言类型：初期聚焦河北方言
   - 语料格式：音频文件（WAV/MP3）+ 文本标注（JSON）
   - 标注内容：`{audio_file, dialect_type, text, speaker_info}`
   - 用途：积累足够语料后用于Whisper微调

5. **文档状态流转**：
   ```
   待处理 → 切片中 → 向量化中 → 已完成
                              → 处理失败（可重试）
   ```

6. **系统监控集成**：知识库文档数量、Chroma向量数量、Embedding API调用量在系统监控模块(模块16)中展示。

7. **申报书对应**：本模块支撑申报书五大研究内容之"知识库+方言AI问答"的知识库基础。
