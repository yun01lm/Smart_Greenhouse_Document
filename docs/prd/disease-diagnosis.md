## 概述

病虫害拍照诊断模块为用户提供"即拍即诊即诊即方"的作物病虫害识别与防治服务。用户通过APP对异常植株拍照上传，系统调用AI图像识别模型自动判定病虫害类型，输出识别置信度，并基于知识库自动生成包含农业防治、物理防治、生物防治、化学防治的完整防治方案。当AI诊断置信度低于阈值时，系统引导用户一键求助平台专家进行人工诊断。

**覆盖端**：APP端（Android）

**使用角色**：
- 棚主(OWNER)：对自己大棚的作物进行病虫害诊断
- 员工(WORKER)：对被授权大棚的作物进行病虫害诊断

## 用户故事

### US-001 拍照上传诊断

> 作为大棚棚主，我发现作物叶片出现异常斑点，希望拍照上传后系统能自动识别是什么病虫害，以便及时采取防治措施。

**验收条件**：
- 支持实时拍照和相册选取两种方式
- 拍照时提供参考框辅助对焦
- 上传前可预览和重新拍摄
- 上传后显示加载状态，诊断完成后展示结果

### US-002 病虫害类型识别与置信度展示

> 作为大棚棚主，我希望看到AI识别出的病虫害类型以及识别可信度，以便判断是否需要进一步确认。

**验收条件**：
- 显示识别结果：病虫害名称、置信度百分比
- 置信度以进度条或星级形式直观展示
- 高置信度(>=80%)显示绿色，中等(70%-80%)显示黄色，低(<70%)显示红色
- 低置信度时自动提示"AI诊断置信度较低，建议求助专家"

### US-003 自动输出防治方案

> 作为大棚棚主，我希望AI不仅告诉我是什么病虫害，还能给出具体的防治方案，以便我立即采取行动。

**验收条件**：
- 方案包含四种防治手段：农业防治、物理防治、生物防治、化学防治
- 每种手段下列出具体操作步骤
- 化学防治方案注明推荐药剂、浓度配比和使用注意事项
- 防治方案来源于知识库，保证信息的专业性和准确性

### US-004 诊断历史记录

> 作为大棚棚主，我希望查看之前的诊断记录，以便追踪作物病虫害历史。

**验收条件**：
- 诊断记录列表按时间倒序排列
- 每条记录显示缩略图、病虫害类型、诊断时间
- 点击可查看完整诊断详情（原图、识别结果、防治方案）
- 支持按大棚筛选诊断记录

### US-005 诊断结果分享与导出

> 作为大棚棚主，我希望将诊断结果分享给其他人或导出保存，以便与他人讨论或记录存档。

**验收条件**：
- 支持生成诊断结果图片（含原图+识别结果+防治方案）
- 支持分享到微信/QQ等社交平台
- 支持保存诊断图片到手机相册
- 支持导出为PDF格式

### US-006 AI低置信度引导求助专家

> 作为大棚棚主，当AI诊断结果不太确定时，我希望系统能引导我直接求助专家，省去自己寻找专家的麻烦。

**验收条件**：
- AI置信度 < 70%时，诊断结果下方显示"求助专家"按钮
- 点击按钮自动跳转到专家咨询模块
- 自动附带诊断图片和AI识别结果作为参考信息
- 如果用户拒绝求助，可继续查看AI给出的防治方案

## 验收标准

### 功能验收

- [ ] 拍照上传功能正常，支持调用系统相机和相册
- [ ] 上传图片大小限制 < 10MB，自动压缩至合适尺寸
- [ ] AI识别响应时间 < 5秒（API调用场景）
- [ ] 置信度显示正确，颜色标识与阈值对应
- [ ] 防治方案包含四种防治手段，内容完整
- [ ] 低置信度(<70%)时"求助专家"按钮可见且可点击
- [ ] 诊断历史记录完整保存，支持回溯查看
- [ ] 分享功能可生成包含诊断结果的图片
- [ ] PDF导出包含完整诊断信息

### 性能验收

- [ ] 图片上传成功率 > 99%
- [ ] AI诊断接口超时时间 < 15秒
- [ ] 诊断历史列表分页加载，每页20条

### AI准确率验收

- [ ] 初期（百度API）：常见病虫害识别准确率 > 80%
- [ ] 后期（ResNet自训练）：目标准确率 > 90%
- [ ] 防治方案关联准确率 100%（正确病虫害→正确方案）

## 备注

1. **AI模型切换方案**：通过策略模式实现 `DiseaseRecognitionProvider` 接口，初期实现 `BaiduAIProvider`，后期实现 `ResNetProvider`。切换只需修改 `application.yml` 中的一行配置。

2. **支持的病虫害类型**（初期）：
   - 真菌性病害：白粉病、霜霉病、灰霉病、炭疽病
   - 细菌性病害：青枯病、软腐病
   - 虫害：蚜虫、白粉虱、红蜘蛛、蓟马
   - 后期根据ResNet训练数据集扩展

3. **防治方案结构**：
   ```
   防治方案 {
     农业防治: [措施1, 措施2, ...]
     物理防治: [措施1, 措施2, ...]
     生物防治: [措施1, 措施2, ...]
     化学防治: {药剂: 名称, 浓度: xx, 注意事项: [...]}
   }
   ```

4. **图片处理规范**：上传图片自动压缩至1024px宽，JPEG格式质量80%，确保API调用效率和流量控制。

5. **诊断记录表结构**：关联user_id、greenhouse_id、crop_cycle_id（可选）、image_url、disease_type、confidence、treatment_plan（JSON）、created_at。

6. **申报书对应**：本模块对应申报书五大研究内容之"视觉AI长势评估与病虫害诊断"的病虫害诊断部分。
