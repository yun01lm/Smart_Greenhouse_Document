## 概述

专家咨询模块为平台引入专家兜底机制，当AI诊断可信度不足或用户遇到复杂技术问题时，可通过APP一键求助平台入驻的农业专家。该模块支持实时文字/图片/视频聊天，支持发送大棚环境快照，以及专家数据授权管理（7天有效期+可撤销）。专家在Web管理端通过专家工作台处理用户求助。

**覆盖端**：APP端（Android）+ Web管理端（专家工作台）

**使用角色**：
- 棚主(OWNER)：发起求助、与专家聊天、管理授权
- 员工(WORKER)：发起求助（被授权功能）、与专家聊天
- 专家(EXPERT)：在Web端专家工作台处理求助

## 用户故事

### US-001 专家列表查看与选择

> 作为大棚棚主，我希望查看平台入驻专家的信息（专业领域、简介），并选择合适的专家发起求助。

**验收条件**：
- 专家列表展示专家头像、姓名、专业领域、在线状态
- 支持按专业领域筛选专家
- 点击专家可查看详细信息（简介、擅长领域、咨询量）
- 在线专家优先展示

### US-002 一键发起求助

> 作为大棚棚主，遇到技术问题需要专家帮助时，我希望一键发起求助，快速创建与专家的对话。

**验收条件**：
- 点击"求助"按钮后，弹出问题描述输入框
- 支持附带图片（最多3张）
- 支持从AI诊断页面直接发起求助（自动附带诊断结果）
- 提交后创建对话并发送首条消息

### US-003 实时聊天 - 文字消息

> 作为大棚棚主，我希望与专家进行类似微信的实时文字聊天。

**验收条件**：
- 对话界面类似即时通讯工具
- 消息实时收发（WebSocket双向通信）
- 显示消息发送时间和已读状态
- 支持消息通知提醒

### US-004 实时聊天 - 图片与视频发送

> 作为大棚棚主，我希望能将作物照片或短视频发送给专家，帮助专家更好地判断问题。

**验收条件**：
- 支持拍照发送和相册选择发送
- 支持录制短视频发送（最长30秒）
- 图片/视频在聊天界面中缩略图展示，点击可放大查看
- 图片/视频发送后显示上传进度

### US-005 发送环境快照

> 作为大棚棚主，我希望一键将当前大棚的环境数据发送给专家，让专家全面了解大棚状况。

**验收条件**：
- 聊天界面提供"发送环境快照"按钮
- 快照内容包含：当前所有传感器数据、24小时趋势图
- 快照以卡片形式展示在聊天界面中
- 专家点击卡片可查看详细数据和趋势图

### US-006 专家授权管理

> 作为大棚棚主，当专家请求查看我大棚的持续数据时，我希望能收到通知并决定是否授权。

**验收条件**：
- 专家发起授权请求后，用户收到通知
- 通知显示专家名称和请求授权的数据范围
- 用户可选择"同意"或"拒绝"
- 同意后可选择授权有效期（默认7天）

### US-007 已授权列表管理

> 作为大棚棚主，我希望查看当前有哪些专家拥有我大棚的数据访问权限，以及授权到期时间。

**验收条件**：
- 授权列表展示专家名称、授权大棚、授权范围、到期时间
- 即将过期的授权（< 24小时）高亮提示
- 已过期授权自动标记为"已过期"
- 支持随时主动撤销授权

### US-008 对话历史记录

> 作为大棚棚主，我希望查看与专家的历史咨询记录，方便回顾专家建议。

**验收条件**：
- 对话列表展示所有历史咨询
- 每条显示专家名称、最近消息摘要、时间
- 点击进入完整对话记录
- 支持按专家筛选

### US-009 专家端 - 求助消息列表

> 作为专家，登录Web端专家工作台后，我希望看到所有待处理的用户求助。

**验收条件**：
- 求助列表按状态分类：待处理、处理中、已结束
- 每条显示用户信息、问题描述、发起时间
- 支持按状态筛选和排序
- 点击进入对话界面

### US-010 专家端 - 实时对话界面

> 作为专家，我希望在Web端与用户进行实时对话。

**验收条件**：
- 对话界面支持文字/图片/视频收发
- 支持查看用户发送的环境快照
- 显示用户在线状态
- 支持结束对话并标记为"已解决"

### US-011 专家端 - 数据授权管理

> 作为专家，当需要查看用户大棚的持续数据时，我希望能发起授权请求并查看已授权的数据。

**验收条件**：
- 对话界面提供"请求数据授权"按钮
- 可选择请求授权的大棚和数据范围
- 查看已授权大棚的实时数据和历史数据
- 查看授权状态（有效/已过期/已撤销）

## 验收标准

### 功能验收

- [ ] 专家列表正确展示平台所有专家及在线状态
- [ ] 一键求助流程通畅：选择专家→输入描述→创建对话→发送消息
- [ ] 实时聊天消息延迟 < 2秒
- [ ] 图片/视频发送和接收功能正常
- [ ] 环境快照内容完整（传感器数据+趋势图）
- [ ] 授权流程完整：请求→通知→同意/拒绝→有效期管理→过期/撤销
- [ ] 已授权列表展示正确，过期和撤销状态更新及时
- [ ] 未读消息有角标数字和通知推送
- [ ] 专家端求助列表按状态正确分类
- [ ] 专家端可正常查看已授权大棚数据

### 性能验收

- [ ] WebSocket消息延迟 < 1秒
- [ ] 图片上传+缩略图生成 < 5秒
- [ ] 视频上传 < 30秒（30秒视频）
- [ ] 环境快照生成 < 3秒

### 安全验收

- [ ] 专家只能查看被授权大棚的数据
- [ ] 授权过期后数据访问自动失效
- [ ] 撤销授权后数据访问立即失效
- [ ] 聊天消息仅对话双方可见

## 备注

1. **专家角色定位**：专家由团队成员或指导老师扮演，属于平台入驻角色。专家通过Web端专家工作台处理用户求助，不直接使用APP。

2. **聊天技术方案**：复用WebSocket（STOMP协议）实现实时聊天，不引入额外的消息中间件（如RabbitMQ）。聊天消息同时持久化到MySQL。

3. **环境快照数据结构**：
   ```
   {
     greenhouse_id, greenhouse_name,
     snapshot_time,
     sensor_data: [{group_id, group_name, parameters: {...}}, ...],
     trend_images: [url_24h_temperature, url_24h_humidity, ...],
     alert_status: {level, count}
   }
   ```

4. **授权生命周期**：
   ```
   专家发起请求 → 用户收到通知 → 用户同意(设置7天有效期)
   → 专家可查看数据 → 7天后自动过期 / 用户主动撤销
   → 授权状态变为已过期/已撤销
   ```

5. **授权数据范围**：专家获得授权后可查看指定大棚的：
   - 实时传感器数据（所有传感器组）
   - 历史数据（授权期间+过去7天）
   - 趋势曲线图
   - 不可控制设备，不可查看其他大棚

6. **专家排班**：初期专家在线状态由专家手动设置（在线/离线），后期可扩展排班功能。

7. **消息通知**：APP端通过WebSocket+系统通知实现消息提醒。用户在线时WebSocket直接推送，离线时通过系统通知推送。
