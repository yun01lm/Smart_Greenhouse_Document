## 五、一键配置切换验证

### 5.1 配置项

```yaml
# application-dev.yml (当前默认 Mock 模式)
ai:
  image:
    provider: mock              # mock | baidu | resnet
  voice:
    provider: mock              # mock | xunfei | whisper
  llm:
    provider: mock              # mock | deepseek
  embedding:
    provider: mock              # mock | siliconflow
weather:
  provider: mock                # mock | qweather
```

### 5.2 切换方式

| 从 → 到 | 操作 | 代码改动 |
|------|------|:--:|
| Mock → 百度 AI | `ai.image.provider: mock` → `baidu` | 零行 |
| Mock → 讯飞 | `ai.voice.provider: mock` → `xunfei` | 零行 |
| Mock → DeepSeek | `ai.llm.provider: mock` → `deepseek` | 零行 |
| Mock → SiliconFlow | `ai.embedding.provider: mock` → `siliconflow` | 零行 |
| Mock → 和风天气 | `weather.provider: mock` → `qweather` | 零行 |

### 5.3 条件装配验证

所有 Provider 均使用 `@ConditionalOnProperty` 注解，Spring Boot 根据配置值自动选择激活的 Bean：

| Provider | 激活条件 | 互斥 |
|------|------|:--:|
| `MockDiseaseRecognitionProvider` | `ai.image.provider=mock` | ✅ 与 Baidu/ResNet 互斥 |
| `BaiduRecognitionProvider` | `ai.image.provider=baidu` | ✅ |
| `ResNetRecognitionProvider` | `ai.image.provider=resnet` | ✅ |
| `MockSpeechRecognitionProvider` | `ai.voice.provider=mock` | ✅ 与 Xunfei/Whisper 互斥 |
| `XunfeiSpeechProvider` | `ai.voice.provider=xunfei` | ✅ |
| `WhisperSpeechProvider` | `ai.voice.provider=whisper` | ✅ |
| `MockLlmProvider` | `ai.llm.provider=mock` (matchIfMissing=true) | ✅ 与 DeepSeek 互斥 |
| `DeepSeekLlmProvider` | `ai.llm.provider=deepseek` | ✅ |
| `MockEmbeddingProvider` | `ai.embedding.provider=mock` (matchIfMissing=true) | ✅ 与 SiliconFlow 互斥 |
| `SiliconFlowEmbeddingProvider` | `ai.embedding.provider=siliconflow` | ✅ |

**结论：一键切换，零代码改动。配置即切换。**
