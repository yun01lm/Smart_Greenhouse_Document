## 三、技术架构快照

### 3.1 后端

| 项目 | 说明 |
|------|------|
| 框架 | Spring Boot 3.x |
| 数据库 | MySQL 8.0（业务数据）+ InfluxDB 2.7（时序数据）+ Chroma（向量） |
| ORM | Spring Data JPA（22 个实体，覆盖 25 张表） |
| 消息 | MQTT（Mosquitto 2.x）|
| 推送 | WebSocket（STOMP 协议）|
| 文件存储 | 本地文件系统（uploads/ 目录）|

### 3.2 前端

| 项目 | 说明 |
|------|------|
| Web 管理端 | Vue 3 + Element Plus，10 个功能页面 |
| APP 端 | UniApp，11 个功能模块 |

### 3.3 AI Provider 架构（策略模式）

| Provider 接口 | 真实实现 | Mock 实现 | 切换方式 |
|------|------|------|------|
| `DiseaseRecognitionProvider` | `BaiduRecognitionProvider` / `ResNetRecognitionProvider` | `MockDiseaseRecognitionProvider` | `ai.image.provider` |
| `SpeechRecognitionProvider` | `XunfeiSpeechProvider` / `WhisperSpeechProvider` | `MockSpeechRecognitionProvider` | `ai.voice.provider` |
| `LlmProvider` | `DeepSeekLlmProvider` | `MockLlmProvider` | `ai.llm.provider` |
| `EmbeddingProvider` | `SiliconFlowEmbeddingProvider` | `MockEmbeddingProvider` | `ai.embedding.provider` |
| 天气 Service | `QWeatherService` | `MockWeatherService` | `weather.provider` |

> **默认配置**：所有 Provider 默认 `mock` 模式，RC 验收开箱即用。切换只需改 YAML，零代码改动。

### 3.4 Docker 基础设施

```yaml
服务:
  - mysql:8.0          # 业务数据库
  - influxdb:2.7       # 时序数据库
  - eclipse-mosquitto:2.x  # MQTT Broker
  - chroma             # 向量数据库
```
